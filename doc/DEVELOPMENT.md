## Folder Rules

One feature = One folder

Reusable UI inside components/ui

Shared layouts inside layouts

Content inside content

Utilities inside utils

---

## Coding Rules

Maximum 150 lines per component.

No duplicated code.

Prefer reusable components.

Use semantic commit messages.

Keep architecture scalable.
