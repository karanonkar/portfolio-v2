# DESIGN.md

# Project Name

Karan Onkar Portfolio

---

# Vision

This is **not a traditional portfolio website**.

The goal is to create a memorable storytelling experience that reflects the journey of a software engineer instead of simply displaying skills and projects.

Every animation, transition, layout, and interaction must support the story.

If an animation does not improve storytelling, it should not exist.

---

# Theme

**From Small-Town Student to Full Stack Engineer**

The website follows a documentary-style experience rather than a standard portfolio layout.

---

# Design Philosophy

The visitor should feel like they are discovering a real person instead of browsing another developer template.

Focus on:

* Storytelling
* Simplicity
* Performance
* Smooth interactions
* Emotional connection
* Authenticity

Avoid:

* Generic portfolio templates
* Hacker UI
* Cyberpunk themes
* Galaxy backgrounds
* Overuse of glassmorphism
* Unnecessary animations
* Random 3D objects

---

# User Experience

The website is divided into chapters instead of sections.

Chapters tell a story.

Example:

↓

Journey

↓

↓

↓

↓

# Design Language

Modern

Minimal

Premium

Documentary Style

Apple-inspired simplicity

Smooth transitions

Readable typography

Dark theme

---

# Animation Rules

Animations must explain something.

Never animate for decoration only.

Use:

* GSAP
* Framer Motion

Animation speed should feel natural.

Avoid excessive motion.

---

# Color Philosophy

Dark background

Minimal accent colors

High readability

No rainbow gradients.

---

# Typography

Large headings.

Short paragraphs.

Lots of whitespace.

Readable hierarchy.

---

# Code Philosophy

Readable code.

Reusable components.

Small functions.

Feature-based architecture.

Avoid unnecessary complexity.

---

# Folder Philosophy

Every feature should remain isolated.

Business logic should never be mixed with UI.

Animations should be reusable.

Content should remain separated from components.

# AI Instructions

If an AI assistant is working on this project, follow these rules.

Do NOT redesign the project.

Do NOT replace the architecture.

Do NOT introduce UI libraries unless requested.

Do NOT change folder structure.

Respect the existing design language.

Always prioritize performance, maintainability, and storytelling.

Suggest improvements only when they align with the project's philosophy.

# End Goal

The final product should feel like a handcrafted experience.

The visitor should remember the journey, not the animations.

The portfolio should reflect growth, discipline, and software craftsmanship rather than visual effects.
